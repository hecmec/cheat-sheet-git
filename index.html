Git est un outil de versionning puissant qui permet d'être sûr de ne jamais perdre ses travaux, plutôt une bonne chose non ? Il permet même de travailler sur des projets tests avant de les fusionner avec le projet final. Informations sur l'auteurs, les modifications, création de branches... Git est presque sans limites pour peu que l'on connaisse ses commandes ! Voilà donc un petit récapitulatif des plus importantes pour travailler correctement.

Les commandes de Git
Commande Action
Commande Action
git init Ammorce le versionning dans le dossier
git status Affiche les fichiers modifiés
git add Ajouter les changements au prochain commit
git commit Créer le commit
git commit -m "votre message" Créer le commit avec un message d'information sur sa nature
git diff Affiches les changements commités
git push origin master Envoie le contenu de la branche master sur Github
git pull origin master Récupère le contenu de la branche master sur Github
git branch votrebranche Crée une branche
git branch -av Liste les branches existantes
git checkout votrebranche Change de branche
git merge votrebranche Fusionne la branche choisie avec la branche actuelle
git reset commit Restaure un commit sans effacer les changements des commits suivants
git blame fichier Montre les changements et leurs auteurs
git remote -v Affiche la liste des dépôts configurés

Pour pourvoir envoyer ses fichiers sur son dépot Github, il faut tout d'abord se connecter à l'aide d'une clef SSH, ce petit tutoriel vous indiquera la marche à suivre. Ensuite une fois la connection établie, il suffit de créer un nouveau dépot dans Github. Au moment de la création, les informations sur les modalités de connexion et notamment les commandes à rentrer dans la console vous serons indiquées. Voir deuxième section: git remote et git push -u origin master.
